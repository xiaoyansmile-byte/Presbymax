# BUG 记录与回归测试台账

## 用途

这个文件用于记录后续发现的零散 bug，并跟踪：

- 问题描述
- 影响范围
- 处理状态
- 修复结果
- 回归测试用例

每新增一个 bug，建议同时补一条对应的回归测试用例，方便后续阶段性做自动化回归。

## 记录规范

建议每条 bug 都包含以下信息：

- 编号
- 发现日期
- 页面 / 模块
- 问题描述
- 复现步骤
- 影响范围
- 当前状态
- 修复方案
- 回归测试用例
- 回归结果

## 状态建议

- `open`：已发现，待处理
- `in_progress`：处理中
- `fixed`：已修复，待验证
- `verified`：已验证通过
- `deferred`：暂缓处理
- `unfix`：已确认问题存在，但当前阶段不修复，留待后续阶段统一处理

## 记录模板

```md
### BUG-001 | <日期> | <模块/页面>

- 状态：open / in_progress / fixed / verified / deferred
- 问题描述：
- 复现步骤：
- 影响范围：
- 期望结果：
- 实际结果：
- 修复方案：
- 回归测试用例：
- 回归结果：
- 备注：
```

### BUG-001 | 2026-04-23 | 训练洞察页未登录可见数据

- 状态：fixed
- 问题描述：未登录状态下，训练洞察页和训练记录接口曾暴露训练数据。

- 实际结果：页面曾显示训练统计和记录
- 修复方案：
  - 页面层增加未登录拦截
  - `/api/training-records` 匿名状态返回空列表

- 回归测试用例：
  - 未登录访问训练洞察页时，只显示登录提示
  - 未登录调用 `/api/training-records` 时返回空数组
  - 未登录调用报告汇总时不返回训练记录
- 回归结果：待自动化覆盖

### BUG-004 | 2026-04-23 | 计划模板修改后今日训练队列未同步

- 状态：fixed
- 问题描述：在管理员计划模板页面修改模板内容后，个人中心和计划概览的模板信息会更新，但今日训练页面的队列仍可能停留在旧的静态列表里。
- 复现步骤：
  1. 打开管理员计划模板页面
  2. 修改当前计划对应模板里的训练项目
  3. 返回个人中心或计划页，观察模板内容已更新
  4. 打开今日训练页面，观察训练队列是否仍显示旧项目
- 影响范围：计划模板管理、今日训练页、计划页、个人中心中的计划展示
- 期望结果：模板改动后，所有读取同一计划模板的页面都应同步显示最新训练项目
- 实际结果：今日训练队列曾使用静态 `todayTrainings`，未随模板变化更新
- 修复方案：
  - 今日训练队列改为按当前计划模板派生
  - 队列状态继续沿用已存状态，新增或删减的项目按最新模板同步
  - 管理员模板页改为 tab 结构，减少多模板编辑时的视觉混乱
- 回归测试用例：
  - 修改当前计划模板后，刷新今日训练页应显示最新训练项目
  - 已完成状态应保留在同名训练项目上
  - 删除模板中的训练项目后，今日训练页不应继续显示该项目
- 回归结果：待自动化覆盖

- 修复方案：已统一训练记录、今日队列和计划完成数的状态源，并增加了防竞态刷新

- 回归结果：已在 Safari 中验证通过

## 新增 bug 时的建议写法

1. 先写清楚复现步骤，尽量短而准
2. 再写影响范围，避免漏掉关联页面
3. 修复后补上回归测试用例
4. 如果暂时不修，也要明确写出原因和后续条件
