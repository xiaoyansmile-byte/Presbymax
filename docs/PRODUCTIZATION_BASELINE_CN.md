## 报告目的

这份文档不是单纯的需求说明，而是对“从 Demo 迁移到产品化工程”这段工作的阶段性总结。

它记录了：

- 哪些关键能力已经落地
- 中途遇到了哪些问题
- 这些问题是如何修复的
- 当前项目已经到达什么基线
- 后续工作从哪里继续

## 项目背景

最初项目是一个浏览器版视觉训练 Demo，使用 HTML / CSS / JavaScript / Canvas / LocalStorage 演示训练流程。

Demo 可以展示训练界面、训练计划、局部统计和一些配置能力，但不具备真正产品所需要的：

- 统一身份认证
- 服务端数据存储
- 角色权限控制
- 管理员配置能力
- 可持续扩展的页面结构
- 可回滚的工程化基线

因此本阶段的目标，是把 Demo 迁移成一个真正可开发、可扩展、可维护的产品工程。

## 遇到的主要问题与处理结果

### 问题 1：SQLite 兼容性和 schema 迁移

在增加用户画像字段后，旧库里没有 `age / gender / surgery_type / surgery_at` 列，导致登录和管理员页面读取时报错。

处理方式：

- 给 SQLite 读取逻辑增加自动补列
- 给初始化脚本增加补列逻辑
- 移除不兼容的 `ALTER TABLE ... IF NOT EXISTS`
- 保证旧库和新库都能继续使用

### 问题 2：训练记录和统计口径不一致

账户页、洞察页、训练页曾经使用不同的训练记录来源和截断规则，导致统计数字不一致。

处理方式：

- 统一训练记录读取逻辑
- 账户页统计改为使用全量记录
- 洞察和报告页从统一接口和统一存储读取

### 问题 3：未登录时数据泄露

训练洞察页曾在未登录时仍能看到训练统计和训练记录。

处理方式：

- 页面层增加登录态兜底
- `/api/training-records` 未登录返回空数组
- 报告汇总在匿名状态下不再吐训练记录

### 问题 4：移动端布局偏移和溢出

在 iPhone / iPad 上，多次出现页面偏移、宽度溢出、卡片不居中、内容被工具栏遮挡的问题。

处理方式：

- 反复修正全局 shell 宽度
- 引入客户端布局调试层
- 调整移动端导航和卡片网格
- 训练页和个人中心页分别做响应式收紧

### 问题 5：训练页交互空间不够

Gabor 训练页在加上倒计时、停止按钮、反馈层和结束总结后，布局几次出现过于拥挤或滚动锚点偏移。

处理方式：

- 将停止按钮整合到倒计时容器内
- 压缩不必要的标题和面板占位
- 调整自动滚动锚点到题目区
- 减少题板边框和格间距，提高有效图案面积

## 当前阶段基线结论

截至目前，这个项目已经完成了从 Demo 到产品工程的第一阶段迁移，形成了可继续开发的稳定基线。

当前基线具备以下特征：

- 有真实的产品工程结构
- 有可运行的训练主流程
- 有管理员配置入口
- 有计划驱动的训练数据模型
- 有本地数据库和迁移脚本
- 有身份认证和权限保护
- 有可持续扩展的页面和路由结构
- 有明确的下一阶段工作清单
