# 视力训练系统（ProsbyMax/MCL）需求功能说明书（前后台分离版）

## 1. 文档目的
基于当前 DEMO（纯前端 + LocalStorage）实现，定义可用于真实项目开发的完整需求基线，指导后续前后台分离、接口设计、数据库设计、测试验收与上线。

## 2. 当前 DEMO 现状（As-Is）

### 2.1 页面与模块
- 首页：训练入口导航（`index.html`）
- 用户管理：注册/登录/个人信息（`auth.html`）
- 训练计划：选择/切换计划（`plans.html`）
- 执行计划：当日训练列表与进度（`execute-plan.html`）
- 六个训练模块：
  - Gabor 配对（`gabor-match-pilot.html`）
  - 闪烁 Gabor 焦点锁定（`flicker-gabor-focus.html`）
  - 对比辨别（`brightness-2afc.html`）
  - 阅读清晰度（`reading-2afc.html`）
  - 眩光/散射可视性（`glare-2afc.html`）
  - 隧道远近切换识别（`tunnel-symbol-speed.html`）
- 数据分析（`analytics.html`）
- 报告生成（`reports.html`）
- 提醒设置（`reminders.html`）
- 数据同步（导入/导出/备份，`sync.html`）
- 管理后台（本地数据查看删除，`admin.html`）

### 2.2 当前核心数据键（LocalStorage）
- `vision-training-users`
- `vision-training-current-user`
- `vision-training-plan`
- `trainingHistory`
- `gabor-training-history`
- `vision-training-reminders`

### 2.3 当前实现痛点（必须在真实项目修复）
- 数据源分裂：计划执行用 `trainingHistory`，分析/报告用 `gabor-training-history`。
- 字段不统一：同为训练记录，时间字段存在 `date` 与 `ts` 两套。
- 用户数据更新存在不一致键：`plans.html` 里有 `users` 与 `vision-training-users` 混用。
- 无后端鉴权与权限控制，管理页可直接访问。
- 密码明文存储（仅 DEMO 可接受，真实项目不可接受）。

## 3. 目标系统（To-Be）

### 3.1 产品目标
- 提供以“计划驱动 + 训练执行 + 数据追踪 + 报告输出”为核心的视觉训练平台。
- 支持多端登录、云端数据一致、可追溯训练记录。
- 支持患者/普通用户与管理员两类角色。

### 3.2 用户角色
- 普通用户（User）：注册登录、选择计划、执行训练、查看分析报告、管理提醒。
- 管理员（Admin）：用户与数据管理、运营看板、数据审计与导出。

## 4. 功能需求

### 4.1 账户与认证
- 用户注册：姓名、邮箱、密码；邮箱唯一校验。
- 用户登录：邮箱+密码；支持 Token 会话。
- 用户登出：服务端会话失效 + 前端状态清理。
- 个人资料：查看/更新昵称、邮箱、注册时间、最近训练时间。
- 安全要求：密码加盐哈希（如 bcrypt/argon2），不允许明文。

### 4.2 训练计划管理
- 计划模板管理（系统内置）：
  - 术后恢复计划
  - 多焦点接触镜适应计划
  - 对比敏感度提升计划
  - 焦点切换强化计划
- 用户选择计划后，生成“用户计划实例”（含开始/结束日期、总次数、训练组合）。
- 支持取消当前计划并重新选择。
- 计划进度按“实际完成训练次数”实时更新。

### 4.3 训练执行
- 执行页展示当日训练列表、完成状态、得分摘要。
- 从计划跳转到具体训练模块，完成后回写训练结果。
- 同一训练每日可配置是否允许重复计分（需后端配置）。
- 训练模块统一记录：开始时间、结束时间、得分、准确率、用时、关键参数。

### 4.4 训练模块（6项）
- Gabor 配对训练
- 闪烁 Gabor 焦点锁定
- 对比辨别
- 阅读清晰度
- 眩光/散射可视性
- 隧道远近切换识别

统一要求：
- 每个训练均需支持开始、过程反馈、结束结果。
- 统一结果结构（见第 6 章）。
- 保留每个训练的个性化指标（如 `maxK`、`excellent/medium/poor` 等）。

### 4.5 分析与报告
- 分析页：
  - 总训练次数、总时长、最高分、平均分
  - 趋势图（得分/难度）
  - 类型分布与历史列表
  - 支持时间范围筛选（7/30/90/全部）
- 报告页：
  - 报告类型：摘要/详细/进度
  - 生成预览
  - 导出 JSON（必需）
  - 导出 PDF（真实项目必须实现，DEMO 为占位）
  - 分享链接（可选，二期）

### 4.6 提醒
- 创建提醒：时间 + 重复规则（每天/工作日/周末）。
- 删除提醒。
- 二期建议：接入系统通知/短信/邮件。

### 4.7 数据同步与备份
- 导出个人数据包。
- 导入数据包（含结构校验、版本兼容）。
- 创建备份。
- 清空数据（高风险操作，二次确认 + 操作日志）。

### 4.8 管理后台
- 用户检索与详情。
- 训练记录查询、筛选、删除（受权限控制）。
- 操作审计日志。
- 数据导出（按用户/时间范围/训练类型）。

## 5. 业务流程

### 5.1 用户主流程
1. 注册/登录
2. 选择训练计划
3. 执行当日训练
4. 查看分析与报告
5. 持续训练并跟踪计划达成

### 5.2 关键状态流转
- 计划状态：`not_started` -> `active` -> `completed` / `cancelled` / `expired`
- 训练记录状态：`started` -> `finished`（失败时 `aborted`）

## 7. 接口需求（REST v1）

### 7.1 认证
- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`
- `POST /api/v1/auth/logout`
- `GET /api/v1/auth/me`

### 7.2 计划
- `GET /api/v1/plan-templates`
- `POST /api/v1/user-plans`（选择计划）
- `GET /api/v1/user-plans/current`
- `PATCH /api/v1/user-plans/{id}/cancel`

### 7.3 训练记录
- `POST /api/v1/training-records`
- `GET /api/v1/training-records`（按时间、类型、计划筛选）
- `GET /api/v1/training-records/stats`

### 7.4 报告
- `POST /api/v1/reports/generate`
- `GET /api/v1/reports/{id}`
- `GET /api/v1/reports/{id}/download?format=json|pdf`

### 7.5 提醒
- `GET /api/v1/reminders`
- `POST /api/v1/reminders`
- `DELETE /api/v1/reminders/{id}`

### 7.6 管理端
- `GET /api/v1/admin/users`
- `GET /api/v1/admin/training-records`
- `DELETE /api/v1/admin/training-records/{id}`
- `GET /api/v1/admin/audit-logs`

## 8. 非功能需求
- 安全：JWT/Session、密码哈希、接口鉴权、RBAC、关键操作审计。
- 性能：
  - 常规接口 P95 < 300ms
  - 列表查询（1万条记录量级）P95 < 800ms
- 可用性：移动端与桌面端适配；训练页交互流畅。
- 可靠性：关键数据写入成功率 >= 99.9%。
- 可观测性：错误日志、接口日志、业务埋点（训练开始/结束/中断）。

## 9. 兼容与迁移需求
- 提供一次性迁移脚本：将 LocalStorage 数据导入后端。
- 映射规则：
  - `trainingHistory` 与 `gabor-training-history` 合并进入 `TrainingRecord`。
  - `date/ts` 统一映射为 `endedAt`。
  - Gabor 历史的 `mode/maxK` 放入 `metrics`。
- 迁移后前端停止直接依赖 LocalStorage 业务数据（仅缓存可保留）。

## 10. 验收标准（UAT）
- 用户可完成“注册->选计划->训练->看报告”全链路。
- 六个训练模块均可产生可查询、可统计的统一训练记录。
- 分析页统计与报告页统计口径一致。
- 管理员仅在授权后可访问管理接口。
- 数据导出、导入、备份流程可用且有错误提示。

## 11. 迭代建议
- V1（必须）：认证、计划、训练记录统一、分析、报告 JSON、提醒、基础管理端。
- V1.5（建议）：PDF 报告、消息通知、更细颗粒度计划进度。
- V2（扩展）：医生端、多租户机构管理、A/B 训练处方、设备数据接入。

### 12.15 管理后台页（`admin.html`）
#### 功能设计
- 浏览本地数据，按类型分组展示。

#### 原型图
```text
+--------------------------------------------------------------------------------+
| 管理操作: [刷新] [清空全部]                                                     |
+--------------------------------------------------------------------------------+
| 分组面板A（用户数据）  [清空此组]                                               |

+--------------------------------------------------------------------------------+
| 分组面板B（训练计划）  [清空此组]                                               |
| 表格: ...                                                                       |
+--------------------------------------------------------------------------------+
| 模态框: 数据详情(JSON)                                           [关闭]          |
+--------------------------------------------------------------------------------+
```
